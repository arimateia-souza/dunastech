## Data Flow

1. **User interacts with Sidebar:** User navigates between the presentation modes or functional sections (e.g., "Apresentação Completa", "Estudo de Caso (B2G)", "Assistente B2C", etc.).
2. **Streamlit reruns script:** Streamlit triggers a script rerun and executes the respective render function (`render_hero`, `render_case_study`, etc.) based on the new sidebar state.
3. **Data Loading (Mock or Real-time):**
   - If rendering B2G / Case Study, pandas mock data is loaded and displayed using Plotly.
   - If rendering Integrations and the user initiates collection, `buscar_dados_instagram` initializes `ApifyClient` using the provided token, calls `apify/instagram-scraper`, extracts posts, and returns the list.
4. **Sentiment & Stats Processing:** The retrieved posts are iterated, simple sentiment analysis is performed via string matching of negative/positive keywords, and metric cards are rendered.

## Technical Debt

- [ ] **Google Gemini API Missing:** The project goals specify a generative AI component, but the code currently lacks integration with `google-generativeai`.
- [ ] **Hardcoded Mock Data:** Destination profiles and municipal metrics are hardcoded in the script rather than stored in structured JSON/CSV files.
- [ ] **Lack of Persistent State:** B2C user ratings and feedback are not persisted across sessions or written to files.
- [ ] **Lack of Modularization:** `app.py` is a single 900-line script containing styles, HTML, business logic, visualizations, and data.
